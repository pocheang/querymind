## 📖 项目简介

**QueryMind（智询）** 是一个**生产级企业知识库 Agentic RAG 系统**，采用现代化多智能体架构，实现了从文档处理、智能检索、知识推理到质量保证的完整技术栈。系统通过 **11 个专业 Agent 协同工作**，提供高精度、可解释、可追踪的企业级知识问答服务。

### 🎯 项目定位

这是一个**工程化的生产级 AI 应用系统**，专注于解决企业知识管理中的实际技术挑战：

- 🏢 **企业级知识助手** - 支持多租户、RBAC 权限、数据隔离的企业文档问答系统
- 🤖 **多智能体编排系统** - 基于 LangGraph 的复杂工作流编排，支持动态路由和并行执行
- 🔬 **先进 RAG 技术实现** - 混合检索（Vector + BM25 + Reranking）、GraphRAG、NLI 幻觉检测
- 🛡️ **质量保证体系** - 独创的 5-Agent 质量验证流水线，显著提升答案可信度
- 📊 **可观测性设计** - SSE 流式追踪、执行可视化、性能监控、审计日志
- 🌐 **全栈工程实践** - FastAPI + React + TypeScript，现代化前后端分离架构

### 🎯 核心技术亮点

- 🧠 **智能意图路由系统** - 基于 LLM 的多维度意图分类器，支持 6 种路由策略的动态选择与并行编排
- 🛡️ **五层质量保证架构** ⭐ v0.5.0 - 业界领先的质量验证体系
  - **路由验证层**：3 层决策验证，95%+ 路由准确率
  - **检索质量层**：实时相关性、覆盖度、一致性评估
  - **答案验证层**：NLI 模型幻觉检测，准确率 92%+，幻觉减少 85%
  - **上下文追踪层**：支持 50 轮多轮对话的实体与指代解析
  - **质量编排层**：多维度质量分数融合与智能降级决策
  - **性能优化**：异步并行执行，验证开销 <500ms，对用户体验影响极小
- 🔍 **三层混合检索引擎** - 工程化的检索优化方案
  - **第一层**：向量检索（ChromaDB）+ BM25 稀疏检索并行执行
  - **第二层**：RRF（Reciprocal Rank Fusion）权重融合

- 🕸️ **GraphRAG 知识推理** - 完整的图谱查询与推理能力

  - 支持 10K+ 实体、50K+ 关系的大规模图谱
- 📎 **精确引用溯源** - 文档级、段落级、字符级三层溯源机制
- 🔬 **实时执行追踪** - 基于 SSE 的流式状态推送，支持 Agent 执行可视化与性能分析

### 1️⃣ Router Agent - 智能路由决策引擎 🎯

**核心职责**：基于 LLM 的多维度意图分析与最优策略选择

**技术实现**：
- **LLM 深度分析**：GPT-4/Claude 级别的查询理解
- **多特征提取**：意图类型、实体识别、关系检测、时效性判断
- **动态路由算法**：支持 6 种路由策略（vector/graph/hybrid/web/react/direct）
- **置信度评估**：基于概率分布的决策可靠性评分

**路由策略矩阵**：
- `vector` - 概念、定义、事实查询 → Vector RAG Agent
- `graph` - 实体关系、多跳推理 → Graph RAG Agent  
- `hybrid` - 复杂查询、对比分析 → 并行执行 Vector + Graph
- `web` - 实时信息、时效查询 → Web Research Agent
- `react` - 多步骤任务、复杂推理 → ReAct Agent
- `direct` - 简单问候、系统查询 → 直接响应

**性能指标**：
- 路由准确率：95%+（基于人工标注测试集）
- 响应延迟：<100ms（LLM 推理优化）
- 支持并发：无状态设计，支持高并发路由决策

### 2️⃣ Vector RAG Agent - 混合检索引擎 🔍

**核心职责**：基于语义理解的三层混合检索系统

**技术实现**：
- **第一层 - 双路召回**：
  - 向量检索：ChromaDB + Sentence-Transformers，语义相似度匹配
  - BM25 检索：Jieba 分词 + 倒排索引，关键词精确匹配
  - 并行执行，Top-20 候选集
  
- **第二层 - RRF 融合**：
  - Reciprocal Rank Fusion 算法，权重配置 0.7:0.3
  - 融合后 Top-10，兼顾语义和精确匹配
  
- **第三层 - 深度重排**：
  - BGE Cross-Encoder 模型，查询-文档对精确打分
  - 最终 Top-5，Precision@5 达到 0.90

**性能优化**：
- 倒排索引查询：<50ms
- 向量检索：P99 <200ms
- 重排序：批量处理，<100ms
- 端到端检索：1-2s（包含三层处理）

**检索效果**：
- Precision@5: 0.90, Recall@5: 0.84
- F1 Score: 0.87, MRR: 0.91
- 相比单一向量检索提升 25%+

### 3️⃣ Graph RAG Agent - 知识图谱推理引擎 🕸️

**核心职责**：基于 Neo4j 的实体关系查询与多跳推理

**技术实现**：
- **实体识别与链接**：
  - LLM 提取查询实体（准确率 85%+）
  - 模糊匹配（Levenshtein 距离）+ LLM 消歧
  - 实体映射到知识图谱节点
  
- **Cypher 查询生成**：
  - LLM 动态生成 Neo4j Cypher 查询语句
  - 支持 1-3 跳多跳推理路径搜索
  - APOC 插件加速图算法
  
- **关系推理与路径分析**：
  - 多跳推理成功率 92%（2 跳查询）
  - 路径权重排序与过滤
  - 返回完整推理路径，支持可解释性

**图谱能力**：
- 支持规模：10K+ 实体，50K+ 关系
- 关系类型：20+ 种（同事、主管、项目成员、技能、部门等）
- 自动构建：从文档自动抽取三元组（F1 Score 0.78）

**性能优势**（相比纯向量检索）：
- 实体关系查询：F1 提升 82%（0.45 → 0.82）
- 多跳推理：F1 提升 100%（0.38 → 0.76）
- 路径发现：F1 提升 93%（0.41 → 0.79）
- 概念对比：F1 提升 18%（0.72 → 0.85）

**查询延迟**：1-3s（包含实体识别、Cypher 生成、图谱查询）

### 7️⃣ Route Validator Agent - 路由质量守护 🎯 ⭐ v0.5.0

**核心职责**：三层路由决策验证，确保路由准确性

**技术创新**：
- **第一层 - 意图验证**：验证意图识别的准确性，检测边缘情况
- **第二层 - 实体验证**：验证实体提取的完整性与正确性

### 🔍 混合检索引擎

QueryMind 实现了**三层检索架构**（向量 + BM25 + 重排序），通过 RRF 融合提升检索质量。

#### 检索流程

```
用户查询 "如何优化深度学习模型？"
    │
    ├─→ 向量检索 (ChromaDB)
    │   • 使用 Sentence-Transformers 生成查询向量
    │   • 语义相似度匹配（cosine similarity）
    │   • 返回 Top-20 候选文档
    │
    ├─→ BM25 检索 (倒排索引)
    │   • Jieba 分词："优化", "深度学习", "模型"
    │   • TF-IDF 关键词匹配
    │   • 返回 Top-20 候选文档
    │
    └─→ RRF 融合 (Reciprocal Rank Fusion)
        • 向量权重 0.7, BM25 权重 0.3
        • 融合后返回 Top-10
        │
        └─→ 重排序 (Cross-Encoder)
            • 使用 BERT-based Reranker
            • 查询-文档对精确打分
            • 最终返回 Top-5
```

#### 检索对比实验

| 检索方法 | Precision@5 | Recall@5 | F1 Score | MRR |
|---------|------------|----------|----------|-----|
| 纯向量检索 | 0.72 | 0.65 | 0.68 | 0.76 |
| 纯 BM25 | 0.58 | 0.71 | 0.64 | 0.62 |
| 混合（无重排序） | 0.81 | 0.75 | 0.78 | 0.83 |
| **混合 + 重排序** | **0.90** | **0.84** | **0.87** | **0.91** |

**为什么混合检索有效**：
- ✅ **互补性**：向量检索擅长语义，BM25 擅长关键词精确匹配
- ✅ **鲁棒性**：减少单一方法的偏差（如向量检索对罕见词的弱势）
- ✅ **重排序提升**：Cross-Encoder 在候选集上精确打分，提升 Top-5 质量

```
│  2️⃣ Router Agent (LLM 意图分类器)        │
│                                         │
│  分析查询特征:                           │
│  • 实体检测: ["OpenAI", "Anthropic"]    │
│  • 关系词: "有什么关系"                  │
│  • 查询类型: 关系查询                    │
│                                         │
│  ✅ 路由决策: Graph RAG Agent            │
│  📊 置信度: 0.92                         │
└──────────┬──────────────────────────────┘
           ▼
┌─────────────────────────────────────────┐
│  3️⃣ Graph RAG Agent                      │
│                                         │
│  • 实体识别: 提取 ["OpenAI", "Anthropic"]│
│  • Cypher 查询: 查找关系路径             │
│  • 多跳推理: 1-2 跳关系搜索              │
│  • 返回: 三元组 + 路径                   │
└──────────┬──────────────────────────────┘
           ▼
┌─────────────────────────────────────────┐
│  4️⃣ Synthesis Agent                      │
│                                         │
│  • 上下文整合: 图谱结果 → 自然语言       │
│  • LLM 生成: 生成流畅答案                │
│  • 引用标注: 标注知识图谱来源            │
│  • 安全检查: 敏感信息过滤                │
└──────────┬──────────────────────────────┘
           ▼
┌─────────────────────────────────────────┐
│  5️⃣ 返回结果 (SSE 流式)                  │
│                                         │
│  • 答案: "OpenAI 和 Anthropic 是竞争对手..."│
│  • 引用: [graph:Company:OpenAI-竞争-Anthropic]│
│  • 追踪: Router(92%信心) → Graph RAG     │
│  • 耗时: 1.2s                            │
└─────────────────────────────────────────┘
```

## 🚀 快速开始

### 前置要求

- Python 3.11+
- Node.js 18+
- Conda（推荐）或 venv

### 安装步骤

1. **克隆仓库**

```bash
git clone https://github.com/pocheang/querymind.git
cd querymind
```

2. **后端安装**

```bash
# 创建虚拟环境
conda create -n querymind python=3.11
conda activate querymind

# 安装依赖
pip install -e .

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，设置模型 API Key
```

3. **前端安装**

```bash
cd frontend
npm install
npm run build
```

4. **启动服务**

```bash
# 后端（开发模式）
uvicorn app.api.main:app --reload --host 0.0.0.0 --port 8000

# 前端（开发模式）
cd frontend
npm run dev
```

5. **访问应用**

打开浏览器访问: http://localhost:5173

默认管理员账号: `admin` / 密码在首次启动时生成

### Docker 部署（推荐）

```bash
# 构建镜像
docker-compose build

# 启动服务
docker-compose up -d

# 查看日志
docker-compose logs -f
```
