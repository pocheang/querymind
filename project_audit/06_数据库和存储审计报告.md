## 2. 发现的问题

### P0级严重问题

#### P0-1: 文档删除不是事务性的
**位置**: `app/api/routes/documents.py:54-75`  
**问题**: 删除文档时依次删除向量、图谱、文件，中间失败会留下孤儿数据  
**影响**: 数据不一致，存储泄漏  
**修复**: 实现两阶段提交或补偿事务  

#### P0-2: Neo4j连接无池化管理
**位置**: `app/graph/neo4j_client.py`  
**问题**: 每次查询创建新连接，高并发下连接耗尽  
**影响**: 连接泄漏，性能下降  
**修复**: 使用连接池  

#### P0-3: SQLite并发写入冲突
**位置**: `app/services/auth/auth_service.py`  
**问题**: SQLite在高并发写入时容易锁超时  
**影响**: 用户操作失败  
**修复**: 使用WAL模式（已启用）+ 增加busy_timeout  

#### P0-4: 向量库无备份机制
**位置**: ChromaDB数据目录  
**问题**: 向量库损坏无法恢复  
**影响**: 数据丢失  
**修复**: 实现定期备份和恢复演练  

### P1级高危问题

#### P1-1: 会话删除后缓存残留
**问题**: 删除会话时未清理query_result_cache  
**影响**: 缓存污染，内存泄漏  
**修复**: 级联清理缓存  

#### P1-2: 图谱节点无引用计数
**问题**: 删除文档后实体节点可能成为孤儿  
**影响**: 图谱污染  
**修复**: 实现引用计数或定期清理  

#### P1-3: 文件存储无容量限制
**问题**: 用户可以无限上传文件  
**影响**: 磁盘耗尽  
**修复**: 实现用户配额和全局容量告警  

#### P1-4: 历史数据无归档策略
**问题**: 会话历史无限增长  
**影响**: 数据库膨胀，查询变慢  
**修复**: 实现冷热分离和自动归档  

#### P1-5: 块存储无去重
**问题**: 相同内容的块重复存储  
**影响**: 存储浪费  
**修复**: 使用内容哈希去重  

#### P1-6: 数据库无健康检查
**问题**: 无法检测数据库连接状态  
**影响**: 故障发现滞后  
**修复**: 实现健康检查端点  

### P2级中等问题

#### P2-1: 无数据一致性巡检
**问题**: 无法检测跨存储数据不一致  
**影响**: 数据质量下降  
**修复**: 实现定期一致性检查  

#### P2-2: 迁移脚本无回滚
**问题**: 迁移失败无法回滚  
**影响**: 数据损坏  
**修复**: 实现迁移回滚机制  

#### P2-3: 无存储使用统计
**问题**: 无法监控各存储使用情况  
**影响**: 容量规划困难  
**修复**: 实现存储监控仪表盘

## 3. 存储架构分析

### 3.1 存储映射关系

```
文档 (document_id)
  ├─ 文件系统: data/docs/{filename}
  ├─ 向量库: ChromaDB collection
  │   └─ 子块: chunks.jsonl
  ├─ 父块存储: parents.jsonl
  ├─ BM25语料: corpus_store
  └─ 图谱: Neo4j
      ├─ Source节点
      ├─ Entity节点
      └─ MENTIONED_IN/RELATED边
```

### 3.2 一致性问题

**问题1**: 删除文档时5个存储需要同步清理  
**问题2**: 无分布式事务保证  
**问题3**: 部分失败无补偿机制  
**问题4**: 无一致性验证工具

## 4. 数据生命周期问题

### 4.1 无限增长的数据

| 数据类型 | 增长速度 | 清理策略 | 风险 |
|---------|---------|---------|------|
| 会话历史 | 高 | 无 | 数据库膨胀 |
| 审计日志 | 中 | 无 | 日志表过大 |
| 向量索引 | 中 | 手动删除 | 存储耗尽 |
| 文件上传 | 高 | 手动删除 | 磁盘耗尽 |
| 缓存数据 | 高 | TTL | 内存耗尽 |

### 4.2 建议策略

1. **会话历史**: 90天后归档到冷存储
2. **审计日志**: 1年后归档
3. **向量索引**: 文档删除时级联清理
4. **文件上传**: 用户配额 + 全局限制
5. **缓存数据**: 已有TTL，增加LRU淘汰
