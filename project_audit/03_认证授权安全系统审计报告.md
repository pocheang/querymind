## 2. 已实现功能清单

### 2.1 认证功能

✅ **用户注册**

- 审计日志记录

✅ **用户登录**

- 登录速率限制（IP + 用户名）
- 失败审计记录

✅ **会话管理**
- 令牌过期机制（可配置TTL，默认24小时）
- 会话自动清理（过期令牌删除）
- 最后活跃时间追踪（last_seen_at）
- 会话计数统计

✅ **用户状态管理**
- 用户状态字段（active/disabled）
- 禁用用户会话立即失效

### 2.2 授权功能

✅ **基于角色的访问控制（RBAC）**
- 三种角色：admin, analyst, viewer
- 权限映射表（_ROLE_ACTIONS）
- 通配符权限支持（admin: "*"）
- 细粒度操作权限（query:run, document:manage_own等）

✅ **管理端安全**
- 自我修改防护（管理员不能修改自己的角色/状态）

✅ **审计日志**
- 哈希链审计日志（防篡改）
- 事件分类（event_category, severity）

### 2.3 数据保护

✅ **密码安全**
- PBKDF2-HMAC-SHA256 哈希算法
- 200,000 迭代次数（符合OWASP建议）
- 随机盐值（16字节）

✅ **敏感数据加密**

- 加密密钥独立管理（.api_settings.key）

✅ **会话安全**

- 令牌长度40字节（240位熵）

- 令牌不在响应中暴露（可配置）

## 3. 发现的问题

### 3.1 严重问题（P0 - Critical）

#### P0-1: 会话令牌无刷新机制
**位置**: `app/services/auth/session_manager.py`  
**问题**: 令牌过期后用户必须重新登录，无refresh token机制  
**影响**: 
- 用户体验差（长时间操作被强制登出）
- 无法实现"记住我"功能
- 无法主动吊销所有设备会话

**触发条件**: 令牌过期（默认24小时）  
**根因**: 只实现了单一access token，未实现refresh token  
**修复建议**:
```python
# 实现双令牌机制
def create_session(self, user_id: str, username: str, role: str, status: str) -> dict[str, Any]:
    access_token = secrets.token_urlsafe(40)
    refresh_token = secrets.token_urlsafe(40)
    access_expires = now() + timedelta(hours=1)  # 短期
    refresh_expires = now() + timedelta(days=30)  # 长期
    # 存储两个令牌...
```
**需要补充测试**: 
- refresh token轮换测试
- 过期refresh token拒绝测试
- 吊销所有会话测试

#### P0-2: 登录速率限制可绕过
**位置**: `app/api/routes/auth.py:45`  
**问题**: 速率限制键为 `login::{ip}::{username}`，攻击者可以通过更换用户名绕过IP限制  
**影响**: 
- 暴力破解防护失效
- 可以枚举有效用户名
- 单个IP可以无限尝试不同用户名

**触发条件**: 攻击者使用同一IP尝试多个不同用户名  
**证据**:
```python
# auth.py:44-45
username_key = (req.username or "").strip().lower() or "unknown"
login_key = f"login::{ip}::{username_key}"
```

**根因**: 速率限制粒度设计不当  
**修复建议**:
```python
# 增加全局IP限制
global_ip_key = f"login_global::{ip}"
if login_limiter.is_limited(global_ip_key, threshold=20, window=3600):
    raise HTTPException(status_code=429, detail="too many login attempts from this IP")

# 保留用户名+IP限制（更严格）
user_ip_key = f"login::{ip}::{username_key}"
if login_limiter.is_limited(user_ip_key, threshold=5, window=900):
    raise HTTPException(status_code=429, detail="too many attempts for this account")
```
**需要补充测试**: 
- 多用户名绕过测试
- 全局IP限制测试
- 分布式攻击模拟

#### P0-3: 管理员审批令牌存储不安全
**位置**: `app/services/auth/auth_service.py:82, 151`  
**问题**: `admin_approval_token_hash` 字段名暗示使用哈希，但实际存储方式未知，可能使用简单哈希（易被彩虹表攻击）  
**影响**: 
- 审批令牌泄露风险

- 单点失陷风险

**触发条件**: 数据库泄露或内部人员攻击  
**根因**: 未使用加盐慢哈希算法  
**修复建议**:
```python
# 使用与密码相同的PBKDF2哈希
def hash_approval_token(token: str) -> tuple[str, str]:
    salt = generate_salt()
    token_hash = hash_password(token, salt, iterations=200_000)
    return salt, token_hash

# 验证时
def verify_approval_token(token: str, salt: str, stored_hash: str) -> bool:
    return verify_password(token, salt, stored_hash)
```

#### P0-5: 无会话并发控制
**位置**: `app/services/auth/session_manager.py`  
**问题**: 同一用户可以创建无限个会话，无并发限制  
**影响**: 
- 账号共享无法检测
- 会话泄露后无法强制下线其他设备
- 资源消耗无上限

**触发条件**: 用户多次登录或账号被盗  
**根因**: 未实现会话数量限制和设备管理  
**修复建议**:
```python
def create_session(self, user_id: str, username: str, role: str, status: str, 
                   max_sessions: int = 5) -> dict[str, Any]:
    # 检查现有会话数
    with self.conn_factory() as conn:
        count = conn.execute(
            "SELECT COUNT(*) as c FROM auth_sessions WHERE user_id=? AND expires_at > ?",
            (user_id, iso(now()))
        ).fetchone()["c"]
        
        if count >= max_sessions:
            # 删除最旧的会话
            conn.execute("""
                DELETE FROM auth_sessions 
                WHERE token IN (
                    SELECT token FROM auth_sessions 
                    WHERE user_id=? 
                    ORDER BY issued_at ASC 
                    LIMIT 1
                )
            """, (user_id,))
```
**需要补充测试**: 
- 会话数量限制测试
- 最旧会话自动清理测试
- 全部会话吊销测试

### 3.2 高危问题（P1 - High）

#### P1-1: RBAC权限模型过于简单
**位置**: `app/services/rbac.py`  
**问题**: 
- 只有3种角色，粒度不足
- analyst和viewer权限完全相同
- 缺少资源级权限控制（如只能访问自己的文档）
- 无法实现细粒度的数据范围控制

**影响**: 
- 无法满足复杂的权限需求
- 多租户场景无法隔离
- 数据泄露风险

**修复建议**:
```python
# 实现ABAC（基于属性的访问控制）
def can(action: str, user: dict[str, Any], resource: dict[str, Any] = None) -> bool:
    role = user.get("role", "viewer")
    
    # 基础角色检查
    if role == "admin":
        return True
    
    # 资源所有权检查
    if action.endswith(":manage_own") and resource:
        return resource.get("owner_id") == user.get("user_id")
    
    # 数据范围检查
    if resource and "data_scope" in user:
        return resource.get("scope") in user["data_scope"]
    
    return action in _ROLE_ACTIONS.get(role, set())
```

```python
def validate_password_strength(password: str) -> None:
    if len(password) < 8:
        raise ValueError("password must be at least 8 characters")
    if not re.search(r'[A-Z]', password):
        raise ValueError("password must contain uppercase letter")
    if not re.search(r'[a-z]', password):
        raise ValueError("password must contain lowercase letter")
    if not re.search(r'[0-9]', password):
        raise ValueError("password must contain digit")
    if not re.search(r'[^A-Za-z0-9]', password):
        raise ValueError("password must contain special character")
```

## 11. 总结

### 11.1 整体评估

认证授权系统具有良好的基础架构和密码安全实现，但在以下方面存在严重不足：
- **会话管理** - 缺少refresh token、并发控制、设备管理
- **攻击防护** - 速率限制可绕过、无MFA、无异常检测
- **权限控制** - RBAC过于简单，无资源级权限
- **审批流程** - 高危操作缺少二次验证
