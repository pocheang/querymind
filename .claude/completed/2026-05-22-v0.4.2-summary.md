# app/api/main.py
@asynccontextmanager
async def lifespan(app: FastAPI):
    # Startup
    logger.info("startup_runtime ...")
    shadow_queue.start()
    
    if settings.auto_ingest_enabled:
        _auto_ingest_thread = threading.Thread(...)
        _auto_ingest_thread.start()
    
    try:
        yield
    finally:
        # Shutdown
        _auto_ingest_stop_event.set()
        if _auto_ingest_thread:
            _auto_ingest_thread.join(timeout=5)
        shadow_queue.stop(timeout=2.0)
        Neo4jClient.close_shared_driver()

app = FastAPI(lifespan=lifespan)
```

def _configure_cors(app_obj: FastAPI, settings_obj) -> None:
    app_env = (settings_obj.app_env or "").strip().lower()
    
    if app_env in ("prod", "production"):
        origins_raw = settings_obj.cors_allow_origins or []
        if "*" in origins_raw:
            raise RuntimeError(
                "CORS_ALLOW_ORIGINS='*' is forbidden in production. "
                "Set APP_ENV=dev or provide explicit allowlist."
            )
```

## 💡 经验教训

### 做得好的地方

1. **分5个独立提交**: 便于review和理解，出问题时容易回滚
2. **行为保持兼容**: 除CORS防护外，所有变更向后兼容
3. **充分测试**: 新增测试覆盖CORS场景，导入阻断测试验证fallback
4. **详细文档**: Release notes完整记录每个提交的变更
5. **渐进式重构**: 只改当前需要的，不过度设计

### 需要改进的地方

1. **应该更早发现遗漏**: _ROUTE_MODULES遗漏应在开发时就检查
2. **测试覆盖不够**: admin端点的NameError应该有测试覆盖
3. **静态检查**: 可以引入mypy等工具提前发现类型问题

### 给未来的建议

1. **建立路由注册检查**: 自动检测是否所有路由模块都在tuple中
2. **增强类型注解**: 使用mypy等工具做静态类型检查
3. **CI中加入导入测试**: 自动验证可选依赖的fallback路径
4. **定期清理**: 每个版本发布前review代码库，清理死代码
5. **可观测性**: 新代码中避免使用`except: pass`
