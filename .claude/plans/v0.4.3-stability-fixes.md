## 🎯 Objectives

Fix critical stability issues in the agent execution tracking system and Self-RAG evaluator that could cause race conditions, memory leaks, and runtime errors in production environments.

---

## 📋 Issues Identified

### Issue #1: Agent Execution Tracker - Thread Safety & Memory Management
**File**: `app/services/agent_execution_tracker.py`  
**Severity**: High (can cause race conditions and memory leaks)

**Problems**:
1. **Coarse-grained locking** (line 76, 100-101): Single `_traces_lock` protects entire dict, creates contention
2. **No automatic cleanup**: TTL-based cleanup exists but never scheduled
3. **Unbounded memory growth**: Traces accumulate without limit
4. **Per-trace locks memory leak**: `defaultdict` grows without cleanup

### Issue #2: Self-RAG Evaluator - Wrong LangChain API
**File**: `app/services/self_rag_evaluator.py`  
**Severity**: Medium (causes runtime errors)

**Problems**:
1. **Wrong method call** (lines 95, 172): Uses `llm.generate()` which is not standard LangChain API
2. **Should use**: `llm.ainvoke()` for async calls (matches LangChain 0.3+ interface)

## 🛠️ Implementation Strategy

### Phase 1: Agent Tracker Core Fixes (Days 1-2)

#### Step 1.1: Fine-Grained Locking ✅
**Goal**: Reduce lock contention by using per-trace locks

**Implementation**:
```python
from collections import defaultdict
from threading import RLock

class AgentExecutionTracker:
    def __init__(self):
        self._traces: Dict[str, ExecutionTrace] = {}
        self._traces_lock = threading.RLock()  # Reentrant for structure ops
        self._trace_locks: Dict[str, RLock] = defaultdict(threading.RLock)
```

**Trade-offs**:
- ✅ Pro: Higher concurrency, less contention
- ✅ Pro: Scales better with many concurrent executions
- ⚠️ Con: Slightly more complex code
- ⚠️ Con: Memory overhead for per-trace locks (addressed by cleanup)

**Status**: ✅ Completed
- Changed `Lock` to `RLock`
- Added `_trace_locks` with defaultdict
- Verified no deadlocks in testing

#### Step 1.2: Automatic Cleanup Scheduling ✅
**Goal**: Integrate cleanup into FastAPI lifespan

**Implementation**:
```python
# In AgentExecutionTracker
async def start_periodic_cleanup(self, interval_seconds: int = 300):
    """Start background cleanup task."""
    self._cleanup_task = asyncio.create_task(self._cleanup_loop(interval_seconds))

async def _cleanup_loop(self, interval_seconds: int):
    """Background task that periodically cleans old traces."""
    while True:
        try:
            await asyncio.sleep(interval_seconds)
            removed = self.cleanup_old_traces()
            
            # Clean orphaned locks
            with self._traces_lock:
                active_ids = set(self._traces.keys())
                orphaned = set(self._trace_locks.keys()) - active_ids
                for orphan_id in orphaned:
                    del self._trace_locks[orphan_id]
        except asyncio.CancelledError:
            break

async def stop_periodic_cleanup(self):
    """Stop background cleanup task."""
    if self._cleanup_task:
        self._cleanup_task.cancel()
        await self._cleanup_task
```

**Integration in app/api/main.py**:
```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    # Startup
    tracker = get_tracker()
    await tracker.start_periodic_cleanup(interval_seconds=300)
    
    try:
        yield
    finally:
        # Shutdown
        await tracker.stop_periodic_cleanup()
```

**Status**: ✅ Completed
- Implemented periodic cleanup loop
- Integrated into FastAPI lifespan
- Added orphaned lock cleanup
- Graceful shutdown handling

#### Step 1.3: Update clear_all_traces() ✅
**Goal**: Clean both traces and locks

**Implementation**:
```python
def clear_all_traces(self) -> None:
    with self._traces_lock:
        self._traces.clear()
        self._trace_locks.clear()  # Also clear locks
    logger.info("Cleared all execution traces")
```

**Status**: ✅ Completed
