## Guidelines

Project documents in this directory should:
- Be specific to this repository and its deployment/operation
- Stay synchronized with current code and processes
- Avoid duplicating content from the root README or internal assistant notes
- Link to active core documents when broader context is needed

## When to Add Documents Here

Use `docs/project/` for:
- Project-specific runbooks
- Repository-specific troubleshooting guides
- Deployment checklists
- Environment-specific operating notes
- Project-specific integration guidance

Do not use this directory for:
- Historical reports (use `docs/archive/`)
- General documentation standards (use `docs/` root)
- Design specifications (use `docs/design/`)
- General development process docs (use `docs/development/`)
