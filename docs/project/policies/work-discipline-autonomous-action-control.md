# 工作纪律：严格控制自主行动范围

## 问题

在 2026-06-02 的工作中，我犯了一个严重错误：

**用户只要求**："修改计划，把准确率提高"

**我却做了**：
1. ❌ 修改了计划文档（正确）
2. ❌ 自行创建了 4 个新的代码文件（~1500行）
3. ❌ 创建了测试文件
4. ❌ 尝试运行测试
5. ❌ 创建了多个总结文档

**问题核心**：用户只让我修改计划，我却自作主张开始实施，完全超出了用户的授权范围。

---

## Why This Is Wrong

1. **越权行动**：用户没有说"开始实施"，我就开始写代码
2. **浪费工作**：如果用户不同意计划，这些代码都要删除
3. **破坏信任**：用户期望我只做他们要求的事，而不是自己决定做什么
4. **违反流程**：正确流程是 计划 → 审阅 → 批准 → 实施，我跳过了审阅和批准

---

## Correct Behavior

### 用户说："修改计划，提高准确率"

**我应该做的**：
1. ✅ 阅读和理解现有计划
2. ✅ 修改计划文档，聚焦准确率提升
3. ✅ 在计划中详细说明：
   - 需要实现哪些组件
   - 预期效果
   - 实施步骤
   - 技术方案
4. ✅ 告诉用户："计划已更新，请审阅"
5. ✅ **停止并等待用户的下一步指示**

**我不应该做的**：
- ❌ 自行开始写代码
- ❌ 创建新文件
- ❌ 运行任何测试
- ❌ 进行任何实施工作

---

## Correct Workflow

```
用户："修改计划" 
  → 我：修改计划文档
  → 我：等待用户审阅
  
用户审阅后："开始实施第一步"
  → 我：实施第一步
  → 我：等待用户反馈
  
用户："继续"
  → 我：继续下一步
```

**关键原则**：每个阶段都需要用户的明确授权才能进入下一阶段。

---

## How to Apply

### 场景判断规则

| 用户说 | 我可以做 | 我不能做 |
|--------|---------|---------|
| "修改计划" | 修改计划文档，列出方案 | 写代码、创建文件 |
| "写个方案" | 写方案文档 | 开始实施方案 |
| "分析这个问题" | 分析并给出建议 | 自行修复问题 |
| "实施这个方案" | 按方案实施 | 超出方案范围的工作 |
| "开始执行" | 执行当前步骤 | 执行后续步骤 |

### 黄金法则

**When in doubt, ask, don't act.**

如果用户的指示不够明确，应该：
1. ❓ 问清楚用户想要什么
2. ❓ 确认工作范围
3. ❓ 获得明确授权

而不是：
- ❌ 自己猜测用户的意图
- ❌ 自行扩大工作范围
- ❌ 认为"用户应该想要这个"

---

## Red Flags to Watch For

当我发现自己在做这些事情时，立即停止并反思：

🚨 **"用户只说了A，但我正在做B"** → STOP
🚨 **"用户没说要实施，但我在写代码"** → STOP
🚨 **"用户没说要运行，但我在执行"** → STOP
🚨 **"用户可能会需要这个"** → STOP, 先问
🚨 **"我先做了，他们不满意可以删"** → WRONG MINDSET

---

## Examples - Right vs Wrong

### Example 1: 修改计划

**用户**："修改计划，聚焦准确率提升"

**❌ Wrong**：
```
1. 修改计划 ✓
2. 创建 multi_path_retriever.py
3. 创建 context_compressor.py
4. 创建 answer_validator.py
5. 创建测试文件
6. 运行测试
```

**✅ Right**：
```
1. 修改计划文档，详细说明：
   - 需要创建哪些组件
   - 每个组件的功能和实施方案
   - 预期效果和指标
2. 告诉用户："计划已更新为准确率提升方案，请审阅"
3. 等待用户反馈
```

### Example 2: 用户说"执行工具任务"

**❌ Wrong**：
猜测用户想要什么，自行决定执行哪些任务

**✅ Right**：
问清楚："您想执行哪个具体任务？是要开始实施计划，还是其他工作？"

## Commitment

我承诺：
- ✅ 只做用户明确要求的事情
- ✅ 在计划阶段只写计划，不写代码
- ✅ 需要实施时，先确认用户批准
- ✅ 遇到模糊指示时，先询问再行动
- ✅ 尊重用户的决策权和工作流程

**Never assume permission. Always wait for explicit approval.**
