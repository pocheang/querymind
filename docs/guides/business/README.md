# 非技术文档中心

欢迎！这里的文档使用通俗易懂的语言，**零代码**，专为管理者、产品经理和非技术人员编写。

---

## 📚 文档导航

### 🎯 快速开始

**如果您是第一次了解这个系统**，建议按以下顺序阅读：

1. **[系统概览](./SYSTEM_OVERVIEW.md)** ⭐ 必读
   - 这是什么系统？
   - 能解决什么问题？
   - 如何工作的？（用日常语言和类比）
   - 大约阅读时间：15-20 分钟

2. **[术语表](./GLOSSARY.md)** ⭐ 推荐
   - 技术词汇的通俗解释
   - 按字母顺序排列，方便查找
   - 遇到不懂的词就来这里查
   - 可作为词典使用，需要时查阅

3. **[功能说明](./FEATURES.md)** 
   - 系统有哪些功能？
   - 每个功能如何使用？
   - 适合谁用？
   - 大约阅读时间：10-15 分钟

4. **[工作原理](./HOW_IT_WORKS.md)**
   - 更深入的技术原理（仍然通俗）
   - 系统如何一步步工作
   - 为什么这样设计？
   - 大约阅读时间：15-20 分钟

## 🎭 按角色选择文档

### 👔 管理者/决策者
**您关心**：投资回报、风险、成本

**推荐阅读**：
1. [系统概览](./SYSTEM_OVERVIEW.md) - 重点看"核心价值"和"投资回报率"部分
2. [功能说明](./FEATURES.md) - 了解能做什么
3. 直接跳到"常见问题"部分

**关键指标**：
- 投资回收期：6-12 个月（中型企业）
- 效率提升：查找信息时间减少 70-80%
- 硬件成本：¥3-12 万（一次性）
- 维护成本：几乎为零（自动运行）

---

### 📊 产品经理/业务负责人
**您关心**：功能、用户体验、业务价值

**推荐阅读**：
1. [系统概览](./SYSTEM_OVERVIEW.md) - 全面了解系统
2. [功能说明](./FEATURES.md) - 详细了解每个功能
3. [工作原理](./HOW_IT_WORKS.md) - 理解技术实现

**重点关注**：
- 使用场景（企业知识管理、客服、研发支持）
- 用户体验（操作简单，像百度搜索）
- 数据安全（本地部署，数据不出公司）

---

### 💼 项目经理/实施负责人
**您关心**：实施计划、时间线、资源需求

**推荐阅读**：
1. [系统概览](./SYSTEM_OVERVIEW.md) - 看"系统需求"和"部署和维护"部分
2. [功能说明](./FEATURES.md) - 了解功能范围
3. [术语表](./GLOSSARY.md) - 与技术团队沟通时参考

**实施时间线**：
- 需求评估：1 周
- 硬件准备：1-2 周
- 系统部署：3-5 天
- 数据导入：1-3 天
- 用户培训：半天
- 试运行：2 周
- **总计**：约 1-1.5 个月

## 📖 文档特点

### ✅ 我们的承诺

- **零代码** - 不会看到任何编程代码
- **通俗易懂** - 用日常语言，避免技术术语
- **丰富类比** - 用生活中的例子解释技术概念
- **视觉辅助** - 大量使用图表、流程图
- **实用导向** - 关注"能做什么"而不是"如何实现"

### 📝 如何使用这些文档

**第一次阅读**：
1. 从头到尾读一遍[系统概览](./SYSTEM_OVERVIEW.md)
2. 浏览[功能说明](./FEATURES.md)
3. 把[术语表](./GLOSSARY.md)收藏，随时查阅

**日常使用**：
- 遇到不懂的技术词汇 → 查[术语表](./GLOSSARY.md)
- 想了解某个功能 → 查[功能说明](./FEATURES.md)
- 需要向他人介绍系统 → 看[系统概览](./SYSTEM_OVERVIEW.md)

**深入学习**：
- 想理解技术原理 → 读[工作原理](./HOW_IT_WORKS.md)

## 💡 提示和建议

### 阅读建议

1. **不要一次读完** - 这些文档信息量较大，建议分几次阅读
2. **结合实际场景** - 想想在您的公司如何应用

### 与技术团队沟通

**有了这些文档，您可以**：
- 理解技术团队在说什么
- 提出更有针对性的需求
- 评估技术方案的合理性
- 与开发人员高效沟通

**建议**：
- 先读非技术文档（本系列）
- 再看技术文档（开发文档）的摘要部分
