# 功能说明 - 系统能做什么

这份文档详细介绍系统的每个功能，用通俗语言解释如何使用。

---

## 📋 功能总览

### 核心功能（每个用户都会用到）
1. **智能问答** - 提问并获得基于文档的答案
2. **文档管理** - 上传、查看、删除文档
3. **会话管理** - 保存和管理对话历史

### 高级功能（提升效率）
4. **流式输出** - 答案逐字显示，无需等待
5. **来源引用** - 答案标注来自哪份文档
6. **连续对话** - 记住上下文，支持追问

### 管理功能（仅管理员）
7. **用户管理** - 添加、删除、管理用户
8. **系统监控** - 查看使用情况和性能指标
9. **模型配置** - 切换和配置 AI 模型

## 1. 智能问答

### 功能描述
**一句话**：像使用百度搜索一样简单，输入问题，获得答案

**详细说明**：
- 在搜索框输入问题
- 系统自动找到相关文档
- 生成清晰的答案
- 附带信息来源

### 使用场景

**场景1：安全威胁查询**
```
问：什么是SQL注入攻击？
答：SQL注入是一种常见的Web应用漏洞，攻击者通过在输入字段中注入
    恶意SQL代码，从而操纵数据库查询...
    
    📌 来源：网络安全基础.pdf 第45页
```

**场景2：应急响应查询**
```
问：发现勒索软件感染后的处置流程是什么？
答：勒索软件应急处置流程：
    1. 立即隔离受感染设备
    2. 保留现场证据和日志
    3. 评估影响范围和数据损失
    4. 启动备份恢复流程
    
    📌 来源：应急响应手册2024.pdf 第18页
```

**场景3：AI技术问题**
```
问：什么是RAG技术？
答：RAG（检索增强生成）是一种结合信息检索和生成式AI的技术。
    它先从知识库中检索相关信息，然后将检索结果提供给大语言模型
    生成答案...
    
    📌 来源：大模型技术白皮书.pdf 第23页
```

### 问题类型

**✅ 擅长回答**：
- **事实性问题** - "SQL注入的原理是什么？"
- **步骤说明** - "如何配置防火墙规则？"
- **对比分析** - "IDS和IPS有什么区别？"
- **概念解释** - "什么是零信任架构？"

**⚠️ 有限支持**：
- **预测性问题** - "未来的网络威胁趋势？"（只能基于已有文档推测）
- **创意性问题** - "帮我设计一套安全架构"（需要创造性思维）

**❌ 不能回答**：
- **文档外问题** - 没有上传相关文档的问题
- **实时信息** - "现在有哪些最新漏洞？"、"今天的威胁情报"

### 提问技巧

**好的提问方式**：
```
✅ "什么是APT攻击？如何检测？"
✅ "Log4j漏洞的利用方式有哪些？"
✅ "如何配置Web应用防火墙？"
```

**不好的提问方式**：
```
❌ "这个"、"那个" - 指代不清
❌ "给我所有安全知识" - 太笼统
❌ "为什么我的系统被攻击了？" - 系统无法访问您的环境
```

**提高准确度**：
- **具体清晰** - "CVE-2021-44228漏洞"比"漏洞"更准确
- **使用关键词** - 包含专业术语如"横向移动"、"权限提升"
- **分步提问** - 复杂攻击链拆分成多个阶段提问

### 查询策略

系统自动为您选择最佳策略，但了解这些概念有助于理解结果：

**快速模式**（简单问题）
- 速度：3-5 秒
- 适合：事实查询、单一概念
- 示例："什么是XSS攻击？"

**平衡模式**（常规问题）⭐ 默认
- 速度：5-10 秒
- 适合：大多数问题
- 示例："SQL注入和XSS有什么区别？"

**深度模式**（复杂问题）
- 速度：10-20 秒
- 适合：多文档综合、复杂分析
- 示例："分析APT组织的典型攻击手法和防御策略"

**深度模式**（复杂问题）
- 速度：10-20 秒
- 适合：多文档综合、复杂分析
- 示例："过去三年的市场趋势分析"

## 2. 文档管理

### 功能描述
**一句话**：上传您的文档，系统自动处理并索引

### 支持的文档类型

**文本文档**：
- ✅ **PDF** - 最常用，推荐格式
- ✅ **Word** (.docx, .doc)
- ✅ **纯文本** (.txt)
- ✅ **Markdown** (.md)

**图片**：
- ✅ **扫描件** - 系统会自动识别文字（OCR）
- ✅ **照片** - 包含文字的图片
- ✅ **截图** - PNG, JPG, JPEG

**限制**：
- 单个文件：最大 20MB
- 单次上传：最多 20 个文件
- 总容量：根据服务器配置（通常数GB到数十GB）

### 上传流程

```
1. 点击"上传文档"按钮
   ↓
2. 选择文件（可多选）
   ↓
3. 系统自动处理：
   - 提取文字内容
   - 识别图片中的文字（如有）
   - 理解文档结构
   - 建立索引
   ↓
4. 显示"处理完成"
   ↓
5. 可以开始提问了！
```

**处理时间**：
- 10页PDF：约1分钟
- 100页PDF：约5-10分钟
- 扫描件（需OCR）：时间翻倍

### 文档列表

**查看已上传的文档**：
- 文件名
- 上传时间
- 文件大小
- 处理状态（处理中/已完成/失败）

**操作**：
- 🔍 **预览** - 查看文档内容
- 📥 **下载** - 下载原文件
- 🗑️ **删除** - 删除文档及其索引

### 文档更新

**如果文档内容变了**：
1. 删除旧文档
2. 上传新版本
3. 系统会重新索引

**建议**：
- 文档命名包含版本号（如"手册_v2.0.pdf"）
- 定期清理过期文档

## 3. 会话管理

### 功能描述
**一句话**：保存您的对话历史，可以随时查看和继续

### 会话概念

**什么是会话**：
- 一次对话的完整记录
- 包含所有问题和答案
- 可以跨天、跨周

**类比**：
- 就像微信的聊天记录
- 每个主题可以开一个新会话

### 创建会话

**自动创建**：
- 第一次提问时自动创建
- 默认名称："会话 2023-12-01"

**手动创建**：
- 点击"新建会话"
- 输入会话名称（如"产品咨询"、"政策查询"）

### 会话操作

**继续对话**：
- 点击会话，自动加载历史
- 可以继续提问
- 系统会记住上下文

**示例 - 连续对话**：
```
问1：2023年的销售额是多少？
答1：2023年总销售额为5000万元

问2：和去年相比呢？
答2：2022年为4200万元，同比增长19%
     [系统理解"去年"指2022年]

问3：增长的主要原因是什么？
答3：根据年度报告，主要原因包括...
     [系统理解这是在追问增长原因]
```

**重命名**：
- 给会话起个有意义的名字
- 方便以后查找

**删除**：
- 删除不需要的会话
- 释放存储空间

**导出**：
- 导出为文本文件
- 可以分享或存档

## 4. 流式输出

### 功能描述
**一句话**：答案像打字一样逐字显示，不用干等

### 工作方式

**传统方式**（其他系统）：
```
提问 → 等待20秒 → 完整答案一次性显示
```

**流式输出**（本系统）：
```
提问 → 1秒后开始显示 → 答案逐字出现
```

### 用户体验

**优势**：
- ✅ **立即反馈** - 1-2秒就能看到答案开始生成
- ✅ **进度感知** - 知道系统在工作，不是卡住了
- ✅ **可提前阅读** - 答案长时，可以边生成边阅读
- ✅ **可随时停止** - 如果答案跑偏，可以中断

**类似**：
- ChatGPT 的输出方式
- 视频的边下边播

### 技术实现

**背后的工作**（您不需要懂，但了解有助于理解）：
1. 系统开始生成答案
2. 每生成几个字就立即发送给您
3. 浏览器收到后马上显示
4. 直到完整答案生成完毕

**SSE技术**：
- Server-Sent Events（服务器推送事件）
- 让服务器主动推送内容到浏览器
- 就像实时新闻推送

## 5. 来源引用

### 功能描述
**一句话**：每个答案都标注来自哪份文档，可点击查看原文

### 引用格式

**答案下方显示**：
```
📌 来源：
  • 产品手册.pdf (第15页)
  • 技术规范.docx (第8-10页)
  • FAQ文档.txt
```

**点击可以**：
- 跳转到原文位置
- 高亮显示相关段落
- 查看完整上下文

### 为什么重要

**可验证性**：
- ✅ 可以核实答案是否准确
- ✅ 了解更多背景信息
- ✅ 引用原文更有说服力

**场景**：
```
场景：向客户解释产品规格
1. 用系统查询规格参数
2. 获得答案和来源
3. 向客户展示原文档截图
4. 更有公信力
```

### 引用类型

**直接引用**：
- 答案直接来自某一段原文
- 引用会标注具体页码

**综合引用**：
- 答案综合了多个文档的信息
- 列出所有参考的文档

**推理引用**：
- 答案基于文档进行了推理
- 标注"基于以下文档推理"

## 6. 连续对话

### 功能描述
**一句话**：像和真人聊天一样，可以追问、深挖

### 上下文记忆

**系统会记住**：
- ✅ 之前讨论的主题
- ✅ 提到的实体（人名、产品名、日期）
- ✅ 对话的逻辑关系

**示例场景 - 网络安全咨询**：
```
问1：介绍一下APT攻击
答1：APT（高级持续性威胁）是一种长期、有针对性的网络攻击...

问2：如何检测？
答2：APT攻击的检测方法包括：1. 异常行为分析 2. 威胁情报关联...
     [理解"检测"指的是检测APT攻击]

问3：有防御方案吗？
答3：防御APT攻击需要多层防护：零信任架构、EDR部署...
     [理解防御是针对APT攻击]

问4：那勒索软件呢？
答4：勒索软件的防御策略略有不同，重点是数据备份和访问控制...
     [切换话题到勒索软件，同时继承检测和防御的讨论主题]
```

### 指代理解

**系统能理解的指代**：
- **它、这个、那个** - 指代之前提到的事物
- **去年、明年** - 相对时间
- **更多、详细、具体** - 要求深入某个话题
- **对比、区别** - 与之前提到的事物对比

**示例**：
```
问：2023年的销售额？
答：5000万元

问：和去年相比？
答：2022年4200万，增长19%
```

### 话题切换

**明确切换**：
```
问：换个话题，关于AI大模型...
```

**自然切换**：
```
问1：SQL注入的原理？
答1：[SQL注入的信息]

问2：XSS攻击呢？
答2：[自动切换到XSS攻击]
```

### 澄清和追问

**追问细节**：
```
问1：勒索软件的防御措施是什么？
答1：[概述防御措施]

问2：更具体一点，备份策略怎么做？
答2：[详细的备份策略信息]
```

**要求澄清**：
```
问：这个防护措施适用于所有场景吗？
答：不完全适用。对于高安全要求的环境，需要额外的...
```

### 功能描述
**一句话**：选择和配置AI模型，平衡性能和成本

### 模型选择

**大语言模型（生成答案）**：

**云端模型**：
```
OpenAI GPT-4
  优点：最强大，答案质量最高
  缺点：按使用量付费，成本较高
  适合：对答案质量要求极高的场景

OpenAI GPT-3.5

Claude (Anthropic)
  优点：长文本处理能力强
  缺点：需要API密钥
  适合：处理长文档
```

**本地模型**：
```
Ollama (Qwen/LLaMA)
  优点：完全免费，数据不出公司
  缺点：需要GPU，质量略低于云端
  适合：高度重视数据安全的企业
```

**嵌入模型（理解文档）**：

```
OpenAI text-embedding-3
  优点：质量高，多语言支持好
  缺点：云端服务，需付费

本地嵌入模型
  优点：免费，数据安全
  缺点：质量略低，需要本地资源
```

**模型设置**：
```
1. 选择模型类型
   - 云端（OpenAI/Anthropic）
   - 本地（Ollama）

2. 填写API密钥（云端）
   或
   选择本地模型

3. 测试连接

4. 保存配置
```

- **超时时间**：查询超时前等待的时间

### 支持的语言

- 英文问题 → 可以引用中文文档

## 常见问题

### Q1: 答案不准确怎么办？
**A**: 可能原因和解决方案：
1. **文档不全** → 补充上传相关文档
2. **问题不清晰** → 重新组织问题，更具体
3. **文档质量差** → 上传清晰版本（避免扫描件模糊）

### Q2: 上传后多久能查询？
**A**: 
- 小文件（<10页）：1-2分钟
- 大文件（>100页）：5-10分钟
- 上传后可以看到处理进度

### Q3: 可以删除问答记录吗？
**A**: 可以
- 删除单条：点击问题旁的删除按钮
- 删除会话：删除整个会话记录
- 不影响文档本身

### Q4: 支持哪些语言？
**A**: 
- 完全支持：中文、英文
- 其他语言：取决于所用的AI模型

### Q5: 手机能用吗？
**A**: 可以
- 浏览器访问即可
- 自动适配屏幕
- 核心功能都支持

### Q6: 能同时多人使用吗？
**A**: 可以
- 支持并发访问
- 具体并发数取决于服务器配置
- 标准配置支持几十到上百人同时使用
