# Chat Workbench Console Refresh Design

## Goal

将主聊天工作台重塑为接近参考图的深色指挥台风格，重点覆盖左侧导航、顶部工具栏、消息区和提问输入区，同时保持现有聊天、会话、文档、Prompt、Agent、上传和设置行为不变。

## Scope

- `frontend/src/pages/ChatPage.tsx`
- `frontend/src/pages/chat/components/ChatSidebar.tsx`
- `frontend/src/pages/chat/components/ChatTopbar.tsx`
- `frontend/src/pages/chat/components/ChatComposer.tsx`
- `frontend/src/pages/chat/components/ChatMessages.tsx`
- `frontend/src/styles/pages.css`

不在本次范围内：

- 认证页视觉调整
- Admin 页面视觉调整
- 后端 API、路由、消息协议或业务逻辑修改
- 新增聊天能力或新的工作流模块

## Chosen Approach

采用“纯样式重塑，少量改 JSX”的方案。

原因：

- 风险最低，能最大程度保留当前 `ChatPage` 的状态管理、事件处理和组件边界。
- 参考图的核心特征主要体现在布局密度、色彩、描边、按钮样式和信息层级，不需要重写整个页面结构。
- 只对局部 DOM 包装和文案层级做增强，就足以让视觉更接近 `Operations Deck / Local RAG` 的控制台感。

## Visual Direction

整体视觉采用“深色指挥台 / 安全运营控制台”方向：

- 大底色为蓝黑与近黑色层级，避免现有通用后台感。
- 主要高亮色使用偏紫电蓝，用于品牌块、激活按钮、描边高亮和标签。
- 卡片使用细描边、低对比底色和轻微发光，不使用厚重阴影。
- 信息层级参考目标图：左栏高密度导航，中部内容留白更克制，底部输入区做成主操作面板。
- 文案标签采用全大写小标题或短英文 kicker，例如 `SESSIONS`、`ASK THE RAG SYSTEM`，增强控制台语气。

## Layout Plan

### Sidebar

- 将左侧栏收敛为更接近参考图的深色窄栏。
- 顶部品牌区改为 `Operations Deck` 风格，保留当前产品语义，但文案和层级更接近参考图。
- 会话区提升为第一优先模块，使用更清晰的选中态、数字徽标和悬停态。
- 工具区不再像普通折叠面板，而是更接近“模块卡片列表”。
- 文档区改为 `Knowledge Base` 语言体系，突出索引数量、待处理数量和上传入口。

### Topbar

- 顶部区域改为紧凑横向控制条。
- 左侧为侧栏开关和 `Local RAG` 品牌信息。
- 右侧整理为若干 pill 按钮，保留现有设置、主题切换、架构入口和用户菜单能力。
- 管理入口若存在，视觉上与普通功能按钮区分，但仍遵循同一套暗色按钮体系。

### Message Area

- 用户消息和助手消息继续分离，但整体边框、底色和圆角更接近参考图。
- 助手消息中的 route、agent、strategy 等元数据标签保留，并强化为控制台标签行。
- 思考摘要、执行过程、引用证据等折叠区统一成深色细边框卡片。
- 空状态改为更像分析台的引导区，而不是泛化的欢迎模块。

### Composer

- 输入区是本次改造的视觉重点。
- 外层做成独立主面板，标题使用 `ASK THE RAG SYSTEM`。
- 文本框采用深色大输入面，发送按钮贴近参考图右下角按钮语义。
- `联网检索`、`推理增强`、`检索策略`、`Agent` 统一成控制条。
- 主 CTA、上传按钮、清空按钮和快捷 Prompt 胶囊都收敛到同一套深色系统里。

## Component Changes

### `ChatSidebar`

- 调整品牌头、分区标题、模块状态文本和局部包裹结构。
- 尽量保留已有子组件调用方式，不改变其数据输入输出。
- 如果需要，仅新增少量语义类名来支持参考图样式。

### `ChatTopbar`

- 调整按钮结构和文案层级。
- 将图标展示从当前 emoji 风格收敛到更克制的文本/符号样式，避免与参考图风格冲突。
- 保持设置、主题、架构、管理、个人资料、改密、登出路径不变。

### `ChatComposer`

- 补充更贴近目标图的包装层级和标签。
- 保持 `Ctrl/Cmd + Enter` 发送、`Esc` 停止、拖拽上传、文件上传和快捷 prompt 行为不变。
- 不修改当前提问逻辑、开关逻辑或 select 值。

### `ChatMessages`

- 调整消息头、标签区、details 卡片外观和空状态文案层级。
- 保持编辑、删除、Markdown 渲染、metadata 展示和折叠逻辑不变。

## CSS Strategy

- 主要在 `frontend/src/styles/pages.css` 中新增或重写聊天页相关选择器。
- 尽量基于现有类名工作，避免引入第二套难维护的样式体系。
- 新增类名仅用于支撑参考图里的品牌块、kicker、状态徽标、控制条和 console 卡片。
- 深色主题样式直接作为主视觉，而不是只靠 `data-theme="dark"` 补丁覆盖。

## Behavior Guarantees

以下行为必须保持不变：

- 会话创建、加载、删除
- 提问、停止生成、清空输入
- `Ctrl/Cmd + Enter` 发送
- 文档上传、拖拽上传、PDF 选择、重建索引、删除文档
- Agent 模式切换
- Prompt 模板保存、编辑、删除、复用
- 设置弹窗、主题切换、登出、个人资料、修改密码、管理页入口

## Risks And Mitigations

- 现有聊天页样式在 `pages.css` 中已有多轮叠加修改，容易出现选择器互相覆盖。
  - 处理方式：优先重写聊天页关键容器和核心组件，不碰 auth/admin 相关选择器。

- 当前文本存在部分编码异常痕迹。
  - 处理方式：改动到的文案直接以 UTF-8 中文或简短英文重写，避免继续复用异常文本。

- 侧栏、顶部栏和输入区都涉及桌面与移动布局。
  - 处理方式：桌面优先贴近参考图，移动端退化为单列和横向换行，但不允许出现遮挡、溢出或不可点击区域。

## Acceptance Criteria

- 聊天页在桌面端视觉上明显接近参考图的深色控制台风格。
- 左栏、顶栏、消息区、输入区形成统一的深色设计语言，不再像多套后台样式拼接。
- 所有现有聊天核心功能继续可用，无明显交互回归。
- 1080px 以下布局仍可用，侧栏抽屉、顶部按钮和输入区不会互相遮挡。
- 改动范围控制在聊天页及其直接样式，不波及 auth/admin 页面。
