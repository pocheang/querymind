# Auth UI Refresh Design

## Goal

将认证相关页面统一为一套接近参考图的浅色企业风登录视觉，覆盖登录页、忘记密码页和修改密码页，同时保持现有认证逻辑、路由和交互行为不变。

## Scope

- `frontend/src/pages/LoginPage.tsx`
- `frontend/src/pages/ForgotPasswordPage.tsx`
- `frontend/src/pages/ChangePasswordPage.tsx`
- `frontend/src/styles/components.css` 中认证页相关样式

不在本次范围内：

- 认证 API 行为变更
- 路由结构变更
- Chat/Admin 等其他业务页样式调整

## Visual Direction

整体风格采用参考图的清爽企业感：

- 页面背景为浅灰蓝渐变，中心放置大圆角白色容器。
- 左栏为浅青蓝渐变品牌介绍区，含 badge、标题、说明、三张功能卡片和柔和装饰圆形。
- 右栏为表单区，强调留白、细边框、轻阴影、蓝色主按钮和浅描边次按钮。
- 移动端下改为上下堆叠布局，保留信息层级，但压缩左右边距和卡片尺寸。

## Page Behavior

### Login

- 保留用户名、密码、记住用户名、忘记密码、登录、注册、Google 登录、GitHub 登录。
- 文案层级和排布向参考图靠拢。
- 输入框加入前缀图标槽位，弱化当前过于“开发工具化”的校验反馈视觉。

### Forgot Password

- 未提交状态使用与登录页同源的双栏骨架。
- 已提交状态仍保留成功说明，但改为同一视觉语言。
- 不新增真实 API，仅沿用当前占位行为。

### Change Password

- 保留密码强度和确认密码校验。
- 左栏改为安全说明卡片，右栏为更接近登录页节奏的表单布局。
- 保留成功后跳转首页的现有行为。

## CSS Strategy

- 沿用现有 `auth-*` 类名体系，避免引入第二套样式架构。
- 在 `components.css` 内重写认证区块样式，新增少量语义类支持图标输入框、介绍卡片、装饰层和信息面板。
- 优先复用已有通用按钮、链接和提示类，减少页面级重复。

## Risks And Mitigations

- 当前仓库存在未提交修改，因此仅在认证页文件和认证样式区域内工作，避免碰到其他业务文件。
- 现有页面文本存在编码异常显示风险，本次会直接用 UTF-8 中文文本重写涉及段落。
- 认证页共用样式较多，需验证三页在桌面和窄屏下都没有布局回归。

## Acceptance Criteria

- 登录页视觉与参考图达到约 90% 一致度，尤其是双栏布局、配色、留白、按钮和社交登录区。
- 忘记密码页和修改密码页共享同一认证设计语言。
- 所有现有交互仍可使用：登录、注册、记住用户名、忘记密码跳转、模拟找回、修改密码提交。
- 移动端不出现溢出、重叠或不可点击区域。
