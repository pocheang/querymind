## 🔧 Configuration Switches

```bash

ENABLE_FAST_RERANK=true              # GPU-accelerated reranking

ENABLE_ADAPTIVE_ROUTING=true         # Auto strategy selection
ENABLE_QUERY_CACHE=true              # 1000-entry cache
```

## ⚠️ Common Gotchas

1. **GPU not detected?** Check: `FAST_RERANKER_DEVICE=cpu` or install CUDA
2. **First run slow?** Model downloads ~400MB (one-time)
3. **Cache not working?** Check: `ENABLE_QUERY_CACHE=true`
4. **Tests failing?** Ensure conda env `rag-local` is activated

## 💡 Pro Tips

1. **Start with "auto" strategy** - let the system optimize for you
2. **Monitor stage timings** - identify bottlenecks quickly
3. **Use cache** - 60% hit rate = 60% faster responses
4. **GPU matters** - 3-5x faster reranking with CUDA
5. **Tune thresholds** - adjust based on your data

## 🆘 Quick Troubleshooting

| Problem | Solution |
|---------|----------|
| Slow retrieval | Check if 3-path is enabled, verify parallel execution |
| Slow reranking | Check GPU is being used (`diagnostics['device']`) |
| Poor results | Try "precise" strategy for complex queries |
| High memory | Reduce `QUERY_CACHE_MAX_SIZE` |
| Import errors | Run `pip install -e ".[reranker]"` |
