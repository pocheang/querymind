## 🔍 现状分析

### 当前代码状态

#### 问题1: 死代码和垃圾文件
```
admin_users.py.backup - 375行重复内容
app/services/auth.py - 135行被子包遮蔽的代码
```

#### 问题2: 静默异常（示例）
```python
# query_result_cache.py
try:
    redis_client.set(key, value)
except Exception:
    pass  # ❌ 静默失败，运维无感知
```

#### 问题3: 过时的FastAPI模式
```python
# main.py
@app.on_event("startup")  # ❌ 已废弃
def startup():
    pass

@app.on_event("shutdown")  # ❌ 已废弃
def shutdown():
    pass
```

#### 问题4: 依赖注入重复
```python
# dependencies.py
from x import y as y_impl

def _y_wrapper():
    from x import y_impl  # 重复导入
    return y_impl(...)

_y = _y_wrapper  # 三重声明
```

### 问题根因

1. **技术债务积累**: 快速迭代导致临时文件未清理
2. **开发习惯**: 过度使用`except: pass`作为快速修复
3. **跟进不及时**: FastAPI升级后未更新最佳实践
4. **打包策略**: 所有依赖默认安装，未考虑使用场景

## 💡 解决方案

### 方案设计 - 5个独立提交

#### Commit 1: 仓库清理 (`ffe46a3`)
**目标**: 删除死代码，整合配置文件

**实施内容**:
- 删除`admin_users.py.backup` (375行)
- 删除`app/services/auth.py` (135行死代码)
- 合并`pytest.ini` → `pyproject.toml [tool.pytest.ini_options]`
- 合并`.coveragerc` → `pyproject.toml [tool.coverage.*]`
- 添加`[tool.ruff]`配置
- 更新`.gitignore`

**影响**: 无（删除未使用代码）

#### Commit 2: FastAPI现代化 + 可观测性 (`6d9d6b0`)
**目标**: 采用lifespan模式，恢复异常可见性

**实施内容**:
```python
# Before
@app.on_event("startup")
def startup():
    pass

# After
@asynccontextmanager
async def lifespan(app: FastAPI):
    # Startup
    pass
    yield
    # Shutdown
    pass

app = FastAPI(lifespan=lifespan)
```

**异常处理改进**:
```python
# Before
except Exception:
    pass

# After
except Exception as e:
    logger.warning(f"Operation failed: {e}", exc_info=True)
```

**修复漏洞**: 补全`_ROUTE_MODULES`缺失的5个路由

**影响**: 无行为变更，仅提升可观测性

#### Commit 3: 简化依赖注入 (`629cb29`)
**目标**: 消除三重声明模式

**实施内容**:
```python
# Before (三重声明)
from x import y as y_impl
def _y_wrapper():
    from x import y_impl
    return y_impl(...)
_y = _y_wrapper

# After (单次声明)
from x import y as y_impl
def y(...):
    return y_impl(..., context_param=...)
```

**影响**: 10个helper简化，无行为变更

#### Commit 4: 生产安全 (`a4c5791`)
**目标**: CORS wildcard拒绝 + 审计日志结构化

**CORS保护**:
```python
def _configure_cors(app, settings):
    if settings.app_env in ("prod", "production"):
        if "*" in settings.cors_allow_origins:
            raise RuntimeError(
                "CORS_ALLOW_ORIGINS='*' is forbidden in production"
            )
```

**审计日志JSON化**:
```python
# Before
detail = f"user={user}; reason={reason}"

# After  
detail = json.dumps({"user": user, "reason": reason})
```

**修复bug**: 捕获`validate_and_check_approval_token()`返回值

**影响**: 生产启动更严格（breaking for misconfigured envs）

#### Commit 5: 可选依赖 (`ceb2810`)
**目标**: 将ML/OCR库设为可选extras

**实施内容**:
```toml
[project]
dependencies = [
  "fastapi>=0.115.0",
  # 移除: paddlepaddle, paddleocr, pytesseract, 
  #      docling, sentence-transformers
]

[project.optional-dependencies]
ocr = ["pytesseract>=0.3.10"]
paddle = ["paddlepaddle", "paddleocr"]
docling = ["docling"]
reranker = ["sentence-transformers"]
full = ["multi-agent-local-rag[ocr,paddle,docling,reranker]"]
```

**降级处理**: 代码已有fallback逻辑
- Reranker缺失 → 词法评分
- Docling缺失 → 返回空列表 + warning
- pytesseract缺失 → OCR status标记
- paddleocr缺失 → 返回None

**影响**: 核心安装减少~2GB

### 技术决策

**为什么分5个提交**:
- 每个提交独立、可review
- 出问题时方便回滚单个提交
- 便于code review理解变更意图

**为什么不添加新功能**:
- 专注于质量提升
- 避免混淆重构和功能
- 为v0.4.3打好基础
